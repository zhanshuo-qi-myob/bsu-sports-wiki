#  投稿文章格式规范
:::tip
推荐使用：
Obsidian撰写你的文章
:::

为了让我们的网站看起来整洁、专业，也为了方便大家阅读，在提交 PR（投稿）之前，请花两分钟看一下这个简单的格式要求。

不用太复杂，主打一个**清晰、易读**。

## 1. 也就是个抬头 (Front Matter)

Docusaurus 需要文章开头有一段特定的配置，不然网页会报错。
请把下面这段代码复制到你文章的**最顶端**，然后修改对应的内容：

```yaml
---
sidebar_position: 0 #这个必须写，代表你的文章索引
id: my-training-log  # 这里的id要是英文，尽量短一点，比如 "leg-day-plan"
title: 深蹲涨力心得分享 # 文章的大标题
---
```

## 2. 标题层级

文章的大标题（H1）系统会自动生成，**正文请直接从二级标题（H2）开始**。

*    正确写法：使用 `##` 开头。
*    错误写法：直接用 `###` 或者纯加粗。

**示范：**

```markdown
## 一、训练前的热身
...内容...

### 1. 髋关节激活
...内容...
```

## 3. 重点与引用

如果是关键的训练要点，或者教练的原话，可以用**引用块**或者**加粗**。

> "深蹲的时候，膝盖内扣是大忌。" —— 某不愿透露姓名的教练

或者：

**记住：核心收紧，不要塌腰！**

## 4. 怎么列训练计划？ (表格与列表)

如果你要分享具体的训练课表，建议用**表格**或者**无序列表**，别堆成一坨文字。

**推荐写法（表格）：**

| 动作 | 组数 | 次数 | 组间休息 |
| :--- | :--- | :--- | :--- |
| 杠铃深蹲 | 5 | 5 | 3min |
| 腿举 | 4 | 10-12 | 90s |
| 坐姿腿屈伸 | 4 | 15 | 60s |

**或者写法（列表）：**

*   **热身**：慢跑 10 分钟
*   **正式组**：
    *   卧推 5x5
    *   推举 4x8
*   **拉伸**：胸大肌放松

## 5. 关于图片

有图有真相。如果你有动作演示图或者比赛照片：

1.  把图片文件放到项目里的 `static/img/` 文件夹下（最好新建一个你的文件夹，比如 `static/img/training/`）。
2.  在文章里这样引入：

```markdown
![深蹲侧面观](/img/training/squat-side.jpg)
```

## 6. 最后的叮嘱

*   **说人话**：尽量通俗易懂，别整太多生僻的学术名词，除非你能在后面解释清楚。
*   **排版**：中英文之间建议留一个空格（比如：使用 GitHub 管理），看着舒服。
*   **内容**：欢迎任何形式的干货，但请确保是你亲身实践过的，或者是权威来源的。
